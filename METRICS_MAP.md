# METRICS MAP (UI → KPI)

## MatchResultCard
- Goal: Result → Request Intro conversion
- Levers:
  - Show path prominently (warmth)
  - Show provenance always (reality)
  - Keep trust details optional (no overwhelm)

## TrustPillsRow
- Goal: Reduce skepticism while staying fast
- KPI: Expand rate 30–60%
- Guardrail: If expand rate >70%, trust is too low or unclear.

## WHY THIS MATCH section
- Goal: Increase Request Intro and Confirm Send
- KPI: Request Intro rate; confirm rate
- Rule: Present like a ledger/system report.

## Intro Friction Preview modal
- Goal: Confirm Send ≥80%
- Rule: Two-line preview, one toggle max.

## North Star (GG-style)
- Warm intro requests sent per active user per week (or per month)
- Core funnel (MVP)
- connect_sources_success_rate
- permissions_pass_rate
- query_to_results_rate
- results_to_request_intro_click_rate
- intro_preview_confirm_rate
- intro_sent_rate

## Drop off reasons
- No path found
- Trust feels unclear
- Social risk too high
- Permissions too scary
- Too much friction
